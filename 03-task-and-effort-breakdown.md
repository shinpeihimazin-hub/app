# フェーズ2: タスク・工数分解の基準

ワークフロー（フェーズ1）の各ステップを、実際に着手できる作業単位（タスク）まで割り、工数を見積もる。

## 分解の粒度の目安

1タスクは「1〜2日以内に完了確認できる単位」まで割る。それより大きい場合はさらに分割する。「RAGパイプラインを作る」は大きすぎる。「ベクトルDBを選定してセットアップする」「文書のチャンク分割ロジックを作る」「Embedding生成バッチを作る」「検索クエリの前処理を作る」まで割る。

## タスク分解表のフォーマット

| # | タスク | 説明 | 依存タスク | 見積工数(h) | 難易度(S/M/L) |
|---|---|---|---|---|---|
| 1 | ... | ... | - | ... | ... |

- **難易度S**: 既知のパターンの組み合わせ、ドキュメント通りに進む
- **難易度M**: 複数コンポーネントの結合、設計判断が要る
- **難易度L**: 前例が少ない、性能・精度のチューニングが必要、外部システムとの調整が要る

## 工数見積りの基準表（目安値）

実際の工数はチームの習熟度・既存資産の有無で大きく変わる。以下は「担当者がフレームワークに慣れている前提」の目安値。初めて使うツールの場合は下記の補正係数を掛ける。

### 設計・準備系

| タスク | 目安工数 |
|---|---|
| 要件ヒアリング・フェーズ0の確定 | 1〜3h |
| ワークフロー設計・Mermaid図作成 | 1〜3h |
| システムプロンプト設計（1エージェント分） | 2〜6h |
| ツール（Function/Tool定義）のスキーマ設計 1個あたり | 1〜3h |

### 実装系

| タスク | 目安工数 |
|---|---|
| 単一エージェントの最小実装（1フレームワークで動く状態） | 4〜8h |
| 外部API連携（認証込み）1個あたり | 3〜8h |
| MCPサーバーの新規実装（ツール2〜3個） | 8〜16h |
| RAGパイプライン構築（チャンク分割・Embedding・検索まで） | 8〜20h |
| ベクトルDBセットアップ・運用設定 | 4〜10h |
| マルチエージェントオーケストレーション構築（3エージェント程度） | 16〜40h |
| Human-in-the-Loop承認フロー実装 | 4〜10h |
| メモリ・会話状態の永続化実装 | 4〜10h |
| ノーコードツール(n8n/Dify等)でのフロー構築 | 2〜8h |

### 品質・運用系

| タスク | 目安工数 |
|---|---|
| ガードレール実装（入出力フィルタ、禁止事項チェック） | 4〜12h |
| 評価用ゴールデンテストセット作成（20〜50件） | 4〜12h |
| 自動評価パイプライン構築 | 6〜16h |
| 可観測性（ログ・トレース）導入 | 4〜8h |
| デプロイ（コンテナ化〜本番投入、環境1つ） | 4〜16h |
| CI/CD構築（テスト自動実行込み） | 4〜10h |
| ドキュメント・引き継ぎ資料作成 | 2〜6h |

## 見積り補正係数

基準値に対して以下を掛け合わせる（複数該当する場合は掛け算ではなく、最大値+残りを0.5倍で加算するなど過剰見積りにならないよう調整する）。

| 要因 | 補正 |
|---|---|
| 担当者がそのツール・フレームワークに不慣れ | +50% |
| 外部API連携（認証・レート制限・仕様変更リスクあり） | +30% |
| 機密情報・セキュリティレビューが必須 | +40% |
| 前例のない性能・精度要件（チューニングに反復が要る） | +50〜100% |
| 複数チーム・複数システムとの調整が要る | +30% |
| 本番運用中のシステムへの組み込み（後方互換性が要る） | +20% |

## クリティカルパスの出し方

1. 依存関係のあるタスクを並べ、依存が無いタスクは並列実行可能とマークする。
2. 依存タスクを順にたどり、最も工数の合計が大きい経路を特定する。
3. その経路がクリティカルパス＝全体の最短完了期間を決める列。ここを短縮しない限り、他のタスクをいくら並列化しても全体は速くならない。

## 出力フォーマット例

```
## タスク分解

| # | タスク | 説明 | 依存 | 見積工数(h) | 難易度 |
|---|---|---|---|---|---|
| 1 | 要件確定 | フェーズ0のヒアリング内容を確定文書化 | - | 2 | S |
| 2 | ワークフロー設計 | Mermaid図と分岐条件の確定 | 1 | 2 | S |
| 3 | システムプロンプト作成 | エージェントの役割・制約を定義 | 2 | 4 | M |
| 4 | ツール定義×2 | 外部API呼び出し用のFunction定義 | 2 | 6 | M |
| 5 | 最小実装 | フレームワーク上で動く状態にする | 3,4 | 8 | M |
| 6 | 評価セット作成 | ゴールデンテスト20件 | 1 | 6 | S |
| 7 | 評価・チューニング | 評価結果を見てプロンプト・ロジック修正 | 5,6 | 8 | L |

合計: 36h
クリティカルパス: 1→2→3→5→7（28h）
補正: タスク4は外部API連携のため+30%（6h→7.8h、四捨五入で8h）
```
