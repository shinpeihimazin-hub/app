# price-hunter-agent

欲しい商品を型番まで突き詰め、複数のECサイトを横断比較して「真の実質最安値」を根拠付きで教えてくれるAIエージェント。第1弾はClaude Codeのスキルとして実装している。

このエージェントの価値は単純な表示価格の比較（価格.comでもできる）ではなく、**楽天SPU・5と0のつく日・お買い物マラソン、Yahoo!ショッピングの5のつく日・PayPay還元、Amazonのポイントアップキャンペーンなど、日本のECポイント制度をユーザー個別の条件まで踏み込んで計算すること**にある。

## 使い方

1. このディレクトリ（`price-hunter-agent/`）で Claude Code を開く
2. `/price-hunt <商品名や欲しいものの説明>` を実行
   - 例: `/price-hunt ソニーのノイズキャンセリングイヤホン WF-1000XM5`
   - 型番が未確定でも可。対話で条件を詰めてから比較に進む
3. まずポイント環境（保有クレジットカード、楽天SPU加入状況、PayPay/Yahoo!利用状況、購入予定日など）を聞かれる
4. Amazon・楽天市場・Yahoo!ショッピング・価格.com・メーカー直販などを横断調査し、ユーザーのポイント環境を踏まえた実質価格の比較表と結論が返ってくる

スキルの中身は [`.claude/skills/price-hunt/SKILL.md`](.claude/skills/price-hunt/SKILL.md)、ポイント制度の知識ベースは [`.claude/skills/price-hunt/points-knowledge.md`](.claude/skills/price-hunt/points-knowledge.md) にある。

## 設計方針

`SKILL.md` に「ポイント環境ヒアリング → 商品特定 → 価格収集 → 実質価格の正規化（ポイント制度を織り込む） → 検証（詐欺サイト・型番違いの排除） → 根拠付き提示」の6ステップを明文化している。1〜2サイトだけ見て終わらせず、複数カテゴリのサイトを横断することを必須にしている。ポイント制度は改定が多いため、`points-knowledge.md` の数値は構造理解の目安とし、実際の判定はWebSearchで当日時点の情報を確認する運用にしている。
