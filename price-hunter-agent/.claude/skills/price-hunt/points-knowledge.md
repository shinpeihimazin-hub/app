# 日本ECポイント制度リファレンス

price-hunt スキルが「実質価格」を計算するときに参照する知識ベース。ポイント制度は頻繁に改定されるため、ここに書いた数値は**目安・構造の説明**として扱い、実際の判定に使う数値は必ずWebSearchで当日時点の最新情報（各モールの公式キャンペーンページ）を確認して裏取りする。数値を確認せずに断定しない。

## なぜポイントを軽視すると判断を誤るか

表示価格が一番安いサイトが、実質価格でも一番安いとは限らない。特に楽天市場・Yahoo!ショッピングは「表示価格はやや高いが、ポイント還元を含めると実質最安」になるケースが多い設計になっている。逆に、ポイントには以下の「割引ではない」性質があるため、額面通りに引き算してはいけない。

- **期間限定ポイント**: 有効期限が短く（1ヶ月程度）、ユーザーが使い切れる保証がない。使い切れる見込みが低いなら実質価値を割り引く（最悪ゼロ査定）。
- **上限あり**: キャンペーンごとに「上限〇〇ポイント」があることが多く、高額商品では実質還元率が表示%より低くなる。
- **エントリー必須**: エントリーしていないと後から申請しても付与されない。ユーザーがエントリー済みかを確認する。
- **重複条件**: 「他社ポイントカードと併用不可」「一部ブランド除外」など対象外条件がある。
- **付与タイミング**: 即時付与ではなく翌月以降になるものは、資金繰り上は「今すぐ使えるお金」ではない。

## 楽天市場（楽天SPU / イベント）

- **SPU（スーパーポイントアッププログラム）**: 楽天カード、楽天モバイル、楽天プレミアムカード、楽天銀行、楽天証券、楽天トラベル、楽天ブックス、楽天Kobo、楽天アプリ経由など、条件を満たすサービスの組み合わせでポイント倍率が積み上がる仕組み。ユーザーがどのサービスに加入しているかで実質還元率が大きく変わるため、必ずヒアリングする。
- **お買い物マラソン / スーパーSALE**: 期間中、購入する「店舗数」に応じてポイント倍率が上がるイベント（例: 2店舗で+1倍、3店舗で+2倍…）。1店舗だけの買い物ではこの倍率は乗らない。
- **5と0のつく日**: 毎月5日・10日・15日・20日・25日・30日に楽天カードで決済すると倍率アップ。
- **月間獲得上限**: SPU全体やキャンペーンには月間の獲得ポイント上限（例: 月7,000ポイント程度）があることが多く、高額商品１点買いだと上限に張り付いて実質還元率が下がる。

## Yahoo!ショッピング / PayPay

- **5のつく日**: 毎月5日・15日・25日にPayPay残高・PayPayカードで支払うと還元率アップ。
- **LYPプレミアム会員**: 会員向けの上乗せ還元がある。
- **ストアクーポン**: 出店者ごとに独自クーポンがあり、モール共通還元とは別枠で重複適用できることが多い。
- **PayPayポイントの使い勝手**: 期間限定ではない「PayPayポイント」であれば現金同等に近く使いやすい。付与形態（通常/期間限定）を確認する。

## Amazon

- **ポイントアップキャンペーン**: エントリー制・上限ありのことが多い。プライム会員限定条件がつくこともある。
- **Amazonギフト券チャージキャンペーン**: ギフト券への事前チャージでポイント上乗せがある時期があり、通常のポイント還元と別に上乗せできる場合がある。
- **タイムセール/クーポン**: プライム会員限定先行タイムセールなど、会員種別で見えている価格が違うことがある。

## クレジットカード還元（共通ベース・複数枚使い分け）

どのサイトで買っても、決済に使うクレジットカードの通常還元率（0.5〜1.8%程度が一般的、カードにより異なる）は必ずベースとして加算する。ユーザーが複数のカードを保有している場合、「いつも同じカードを使う」前提にせず、比較対象ストアごとに保有カードの中から最も還元率が高いものを選び直す。

保有カードとストアの相性の例（実際の還元率・対象条件は改定されるため必ずWebSearchで確認する）:
- 楽天カード → 楽天市場（SPUの主要な加算要素でもある）
- PayPayカード → Yahoo!ショッピング/PayPayモール（PayPayポイント還元と重なる）
- イオンカード → イオン系列のオンラインストア
- ビューカード（JR東日本） → Suicaチャージ経由の買い物、ルミネ等の提携先
- au PAYカード → au PAY マーケット
- エポスカード、三井住友カード等 → 提携先が個別にあるため、対象ストアが決まってから特約店還元の有無を確認する

同じ買い物でも「モールのポイント制度と相性が良いカード」と「カード自体の特約還元が高いカード」が食い違うことがあるため、両方を計算した上で高い方を採用する。

## ポイントサイト経由（ポイ活ルート）

モッピー、ハピタス、ECナビ、ポイントインカム、ポイントタウン、ちょびリッチ、楽天リーベイツ（Rakuten Rebates）などの「ポイントサイト」を経由してAmazon・楽天市場・Yahoo!ショッピング等の対象ストアにアクセスしてから購入すると、モール側のポイントに加えてポイントサイト独自の還元（通常0.5〜数%、季節キャンペーンでは数十%になることもある）が上乗せされる。実質価格をさらに下げられる可能性がある最後の一段として、必ず候補に入れる。

計算・提示時の注意:
- ユーザーがそのポイントサイトのアカウントを持っているか（未登録なら候補から外すか「新規登録すれば使える」と注記に留める）
- 経由先モールが実際にそのポイントサイトの対象ストアになっているか（対象外カテゴリ・対象外店舗があるため、いつも同率とは限らない）
- 経由率・還元率はキャンペーンで頻繁に変動するため、必ずWebSearchで当日時点の掲載率を確認する
- 同一ブラウザセッションで「ポイントサイトのリンクをクリック→そのままモールで購入」という手順を踏まないと反映されないため、購入手順として「まずこのポイントサイトのリンクを踏んでから買う」と明示する
- 成果条件（購入確定から加算までの期間、キャンセル時の取消、否認リスク）があることをユーザーに伝える
- 複数のポイントサイトが同じストアを扱っている場合は、還元率が一番高いものを提案する

これを織り込んだ最終的な実質価格は次の通り。

実質価格（ポイ活込み） = モール実質価格（本体価格＋送料－通常ポイント－上乗せキャンペーン－クーポン－クレカ還元） − ポイントサイト経由分の還元

## ヒアリング項目（実質価格計算の前提として必須）

price-hunt のステップ0でユーザーに確認する項目:

1. 主に使うクレジットカード（還元率、特約店の有無）
2. 楽天会員か／SPU対象サービス（楽天モバイル・楽天カード・楽天銀行など）の加入状況
3. Yahoo!ショッピング・PayPayの利用有無、LYPプレミアム加入有無
4. 今月すでに各モールのポイント上限にどれくらい到達しているか（わかる範囲でよい）
5. 購入予定日（5と0のつく日、お買い物マラソン開催中かなど、タイミングで還元率が変わるため）
6. 獲得ポイントを実際に使い切れる見込みがあるか（期間限定ポイントの実質価値判定に使う）
7. アカウントを持っているポイントサイト（モッピー、ハピタス、ECナビ、ポイントインカム、楽天リーベイツなど）

これらが未回答のままだと、実質価格の計算は「一般的な還元率での概算」であることを明示し、確定額として断定しない。
